PR #6510 — visual regression review

chore(website): migrate tailwind to v4, daisyui to v5 and flowbite to v0.12

Method: Headless Chrome screenshots of ~17 pages plus 5 interactive states on main.loculus.org (baseline) vs worktree-tailwind-v4-fres.loculus.org (PR preview), diffed pixel-by-pixel and reviewed by hand. Caveat: the two environments have different sequence data, so list/table contents differ even when the chrome is unchanged — judgements below are about layout, sizing and styling, not row contents.

TL;DR: Migration is clean. No layout breakage, no missing features. Four small cosmetic differences worth a glance, and one apparent regression (the Keycloak login page) that is actually a preview-environment problem, not a PR regression.

Findings

minor / cosmetic

1. LAPIS query-engine buttons are shorter on the API docs page

On /api-documentation the pill buttons in the "LAPIS query engines" section have less vertical padding in the PR. The "View backend API documentation" button is also slightly smaller. Looks like a DaisyUI v5 default-size change for .btn. Cosmetic; nothing is broken.

MAINPR
minor / cosmetic

2. "Organisms" dropdown trigger gets a more prominent border

In the desktop header, the closed-state "Organisms ▾" button now shows a visible border / button outline. Probably a Tailwind v4 outline/focus-ring change.

MAINPR
minor / cosmetic

3. Mobile sandwich menu items are more spaced out

At 390 px width, the rows in the off-canvas mobile menu have more vertical spacing in the PR, making the menu noticeably taller. Backdrop overlay still renders correctly (the bg-gray-800/50 Tailwind v4 syntax in OffCanvasOverlay.tsx works).

MAINPR
minor / cosmetic

4. Download dialog radio buttons restyled

On the search page → "Download all entries" dialog, the radio-button visuals changed: MAIN shows a recessed inner dot; PR shows a flat filled blue circle with a small white centre. More notable, the disabled child radios (FASTA header style → Accession / Display name) render almost invisibly on PR vs visible-but-greyed on MAIN — worth checking they still convey "disabled but available once parent is selected".

Top: MAIN  ·  Bottom: PR
not a PR regression

5. /user shows "We are sorry — An internal server error has occurred"

Visiting /user on the PR preview lands on the Keycloak host (authentication-worktree-tailwind-v4-fres.loculus.org) which is serving an error page instead of the login form. That's the preview's auth deployment, not anything in this PR's website code — but flagging so it doesn't get mistaken for a regression caused by the migration.
